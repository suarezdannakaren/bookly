# Bookly

**User Login**

As a reader, I want to log in with my email address and password so that I can access my account, view my books, and save my favorite books from any device.

*User Login – Acceptance Criteria*

The user has the option to log in.
The system does not allow access if any required field is empty.
The session remains active while the user is using the application and ends when the user logs out.
The system validates that the entered credentials are correct.
⸻

**User Registration**

As a new reader, I want to create an account by providing my personal information (name, email, and password) so that I can access the platform and save the books I have read and my favorite books.

*Acceptance Criteria*

The system verifies that the email address has not been previously registered.
The password must meet the minimum security requirements.
The user can register using their name, email, and password.
⸻

**CRUD (Book Management)**

As a reader, I want to manage my lists of read books and favorite books so that I can keep my information organized and up to date.

*Acceptance Criteri*

The user can add a book to a list.
The user can view the list of registered books.
The user can delete a book from the list.
